Reviewing and updating risks and tests

Your learning journey

In this interactive learning experience, you'll discover:

  • How to spot risks when software changes
  • How to adapt your testing as features evolve
  • How to use RCRCRC for smarter regression testing
  • How to keep your tests relevant and valuable

🎯 Before you start

For the best learning experience:

Viewing options:

  • Option 1: Watch the complete demonstration first, then go through this activity.
  • Option 2: Pause the demonstration after each section to try the activities yourself.

This will help you focus better and avoid missing important information.

Prepare yourself:

  • Have a place to capture notes and thoughts available
  • Try identifying risks before viewing solutions
  • Compare your thoughts with provided examples
  • Think about similar scenarios in your projects

Learning steps

1 Explore the task manager requirement (v1.0)
🤔 What risks would you expect in a simple task system?
 2 Try the implementation (v1.0)
🔍 What potential issues do you notice while using it?
 3 Perform risk analysis (v1.0)
✨ How do your identified risks compare?
 4 Review new priority feature requirements (v2.0)
💭 How might this affect existing functionality?
 5 Experience the enhanced version (v2.0)
📝 List new risks this feature might introduce
 6 Apply the RCRCRC framework for updating tests (v2.0)
🎯 Try applying this to your current work

💡 Practice & self-check

After each step, ask yourself:

  • Can I explain the risks I identified and categorized?
  • How would this apply to my current project?
  • What other risks might exist?
  • How would I test for these risks?
