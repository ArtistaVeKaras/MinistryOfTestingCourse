# Make a DATE Before You Test: A Pre-Testing Checklist for Exploratory Sessions

Before you begin your exploratory testing session, use the **DATE** checklist to ensure you're ready:  
**D**ocumentation, **A**ccess, **T**est data, and **E**nvironment.

---

## D - Documentation Review

*Understand what you’re testing and how you’ll record your findings.*

- [ ] Test requirements reviewed and understood
- [ ] Test cases or charters prepared
- [ ] Test data requirements identified
- [ ] Reporting templates available

## A - Access and Tools

*Confirm you can access the system and have the tools ready.*

- [ ] All necessary access permissions granted
- [ ] Required tools installed and configured
- [ ] Monitoring systems available
- [ ] Communication channels established

## T - Test Data Preparation

*Check that your data is appropriate and ready for testing.*

- [ ] Test data created or identified
- [ ] Data meets testing requirements
- [ ] Sensitive data handled appropriately
- [ ] Data reset procedures in place

## E - Environment Verification

*Ensure you're testing in the correct environment and everything is working.*

- [ ] Correct version deployed
- [ ] Environment is stable
- [ ] Required services running
- [ ] Test data loaded

---

## Before Starting

- [ ] Correct version identified
- [ ] Environment verified
- [ ] Access confirmed
- [ ] Tools ready
- [ ] Test data prepared
- [ ] Objectives clear
- [ ] Documentation ready

## During Testing

- [ ] Notes being taken
- [ ] Issues documented
- [ ] Progress tracked
- [ ] Time managed
- [ ] Blockers communicated

## After Testing

- [ ] Results documented
- [ ] Issues reported
- [ ] Environment state noted
- [ ] Follow-up items identified
- [ ] Lessons learned captured
